FC3 Introduction

Third-order force constants, usually shortened to FC3, quantify the leading anharmonic coupling beyond the harmonic force-constant picture. In a harmonic or FC2-like response, displacing one atomic degree of freedom changes the force on another degree of freedom in an approximately linear way. FC3 measures how that force response itself changes when a second displacement degree of freedom is introduced.

Φijkαβγ = ∂3U ∂uiα ∂ujβ ∂ukγ = - ∂2Fiα ∂ujβ ∂ukγ

The two concept illustrations use the same idea. The first panel sketches a two-body response: one atom is displaced and a force response appears on a neighboring atom. The second panel adds another small displacement in a graphene-like honeycomb network; the force response on the same atom is modified by the coupled pair of displacements. This mixed response is the physical content represented by an FC3 block.

In ShengBTE-type workflows, FC3 enters three-phonon matrix elements and therefore affects phonon scattering rates, phonon lifetimes, and lattice thermal conductivity. The norm of an FC3 block is useful for visual inspection and consistency checking, but it should not be read as a standalone predictor of kappa.

Explorer Purpose

FC3 Visual Explorer converts a dense third-order force-constant file into inspectable geometry and compact analysis figures. It is intended for researchers who need to examine interaction range, dominant and weak FC3 blocks, tensor-component distributions, and distribution-level differences between several force-constant datasets.

The app links each FC3 block to a three-atom triangle, maps block Frobenius norm to color and line width, and provides plots that make cutoff, supercell, displacement, and dataset-comparison effects easier to inspect before the data are used in a thermal-transport calculation.

Required Data

The main input is a FORCE_CONSTANTS_3RD file generated by thirdorder or a ShengBTE-compatible workflow. Each block should provide a block id, Rj and Rk periodic translations, primitive-cell atom labels i, j, k, and 27 Cartesian tensor components.

A Quantum ESPRESSO pw.x structure file is required to reconstruct lattice vectors, species, and basis positions. The parser expects CELL_PARAMETERS and ATOMIC_POSITIONS crystal. Compatible primitive cells and regular repeated supercells are supported; for supercells, the app infers the primitive basis when possible.

Operation And Calculations

Use Data Input on the analysis pages to load a new FC3 file and a matching QE structure. The embedded example opens automatically. The 3D page displays atoms, bonds, cell boundaries, and FC3 triangles. Boundary inputs choose the visible lattice-image range; Norm range filters FC3 blocks by Frobenius norm; View Direction and zoom controls adjust the camera; the Objects panel hides or highlights elements, atom labels, and individual blocks.

Pick atom lets you click a visible atom instance and emphasize FC3 blocks that include that specific atom image. Isolate picked atom hides unrelated FC3 network lines so the selected atom's three-body interactions can be inspected without visual clutter.

For each block, the plotted triangle is built from xi = ri, xj = rj + Rj, and xk = rk + Rk. The scalar strength is the Frobenius norm over the 27 Cartesian tensor components of that FC3 block:

FC3_block = ||Φblock||F = √ Σαβγ (Φijkαβγ)2

The component distribution counts individual tensor values; the block distribution counts FC3 block norms; the perimeter plot uses the sum of the three triangle edges; the max-edge plot uses the longest triangle edge. Compare overlays the same descriptors for up to four FC3 files against one structure and does not assume identical block counts.
